Problem Statement

UPI fraud often occurs due to social engineering, where users unknowingly authorize transactions. Traditional fraud detection systems fail to detect behavioral anomalies in real-time and only act after the fraud has occurred.

TrustPay solves this by intervening before the transaction is completed — analyzing behavioral deviations and identifying possible social engineering influence in real time.

Objectives
  • Model normal user transaction behavior using statistical behavioral fingerprinting
  • Detect unusual transaction patterns using Z-score anomaly detection
  • Identify high-risk transactions in real-time with fraud probability scoring (0–100)
  • Provide contextual warnings with explainable AI reasoning per anomaly
  • Reduce social engineering risks through keyword detection and scam pattern matching
  • Visualize user behavior patterns through an interactive analytics dashboard

System Architecture
1
User Initiates Transaction — Enters UPI ID, amount, and note in the TrustPay app
↓
2
Behavioral Fingerprint Check — System builds statistical model from user's 200 most recent transactions (avg, median, std deviation, known receivers, active hours)
↓
3
Real-Time Anomaly Detection — 8 independent anomaly checks run simultaneously against the behavioral fingerprint
↓
4
Fraud Probability Score — Weighted combination of anomaly scores (severity weights: LOW=0.6, MEDIUM=0.8, HIGH=1.0, CRITICAL=1.3) → 0–100 score with confidence level
↓
5
Smart Warning UI — Contextual alert with per-anomaly evidence, smart protective question, and recommended action (CANCEL / VERIFY / SAFE)
↓
6
User Decision + PIN Confirmation — User cancels or proceeds with PIN verification. Cancelled high-risk transactions are logged to fraud prevention records
Behavioral Anomaly Detection Model
A1 — New Receiver
Sending to a UPI ID not in transaction history
Score: +25 | Severity: HIGH
A2 — Amount Deviation
Z-score based: amount vs personal average (extreme: 10x+)
Score: +10 to +35 | Severity: MEDIUM–CRITICAL
A3 — Time Anomaly
Transaction at unusual hour (before 6AM / after 11PM)
Score: +10 to +20 | Severity: MEDIUM–HIGH
A4 — Rapid Transactions
3+ transactions in last 10 minutes
Score: +12 to +20 | Severity: MEDIUM–CRITICAL
A5 — Social Engineering
30+ urgency/scam keywords detected in note field
Score: +28 | Severity: CRITICAL
A6 — Scam Pattern
5 pattern categories: impersonation, lottery, urgency, threat, investment
Score: +18 | Severity: HIGH
A7 — Round Number
Large round-number amounts (≥₹10,000 divisible by 1000)
Score: +5 | Severity: LOW
A8 — New User High Amount
High amount from account with fewer than 5 transactions
Score: +15 | Severity: HIGH
